Personal Website

  • Java
  • Typescript
  • Docker
  • AWS CDK
  • Spring Boot
  • Web-dev

Hosting my personal website was a fantastic exercise allowing me to explore my interest in web development. The webpage is hosted on AWS, with traffic being passed through an Application Load Balancer and distributed to Docker containers running in an ECS Fargate cluster.

What I did:

  • Developed backend-API logic using the Spring Boot Java framework
  • Deployed hosted infrastructure via AWS's Typescript Cloud Development Kit
  • Utilized cloud resources effectively by deploying website Docker image to self-scaling containers via AWS ECS Fargate
  • Purchased custom domain name and configured AWS Route 53 records and SSL certificates to enable both HTTP and HTTPS connections to the ECS cluster load balancer.

Please note the following:

  • The decision to host through this webpage through ECS Fargate was a conscious decision. While this website could have been stored and hosted in an S3 bucket, I elected for a more complicated, (and to be quite frank, overkill) design for the sake of experience.
  • This project is not meant to highlight proficiency in JS, CSS, or HTML. The styling of this website was sourced via this webpage created by Eina Onting.

Virtual Memory Management

  • Operating-Systems
  • Memory-Management
  • Linux
  • Algorithms

This project was my submission for a graduate-level Operating Systems course taken at NYU Courant. The project focused on implementing virtual address spaces, and mapping said spaces from virtual pages to physical disk frames. Several popular page-replacement algorithms were implemented including “Last Recently Used”, “Aging”, and “Working Set”.

What I did:

  • Utilized Object-Oriented design principles to implement six different virtual page mapping algorithms
  • Leveraged C++ features such as struct Bitfields, classes, virtual functions, and pointers to solve tasks in a memory efficient manner
  • Followed best practices utilizing Git for version control, makefile for code build reproducibility, and enumerated types for legibility

Process Scheduler

  • C++
  • Operating-Systems
  • Discrete-Event-Simulation
  • Linux
  • Algorithms

This project was my submission for a graduate-level Operating Systems course taken at NYU Courant. The project focused on implementing process scheduling algorithms, and simulating their relative performance via a discrete-event simulation. Scheduling algorithms performance was ranked by cpu utilization, average time for process completion, and average process turn-around time. Several popular page-replacement algorithms were implemented including “Round Robin”, “Priority Queing”, and “Preemptive Priority Queing”.

What I did:

  • Wrote 1700 lines of C++ source code, earning a perfect score of 248/248 test cases passed
  • Utilized Object-Oriented design principles and C++ Container STL library to implement six process scheduling algorithms via dynamic-dispatch
  • Wrote custom implementation for Priority-Queue data structure
  • Followed best practices utilizing Git for version control, makefile for code build reproducibility, and enumerated types for legibility
